有意义的命名

​     名副其实，能封装就封装，小函数没啥

​     名称的区分度

​     可搜索的名称

​     名称编码？ 匈牙利命名法？成员前缀？接口编程？ 减少名称的信息，上下文足够分辨出来

​     思维映射，避免覆盖导致歧义，混乱记忆，双关的更不建议

​     类名称不要是动词，方法动名结合

​     一个概念就用一个词，一会get 一会fetch，一会controller一会manager

​    恰当有意义的语境，使用领域内的名称，不用没意义的语境
